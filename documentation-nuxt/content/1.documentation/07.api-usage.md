## How It Works

::callout{icon="i-lucide-info" color="warning"}
There are no real API requests to a server. The application reads data directly from these static JSON files. You can modify or extend them as needed.
::

To use real data instead of mock JSON files, create reusable API helpers and fetch data with React hooks. This keeps components clean and allows easy switching to real REST endpoints.

## Integration Steps

::steps

### Create a Product Data Hook

Create a custom React hook to fetch products with axios:

```ts [src/hooks/useProducts.tsx]
import { useEffect, useState } from "react";
import axios from "axios";

export const useProducts = () => {
  const [products, setProducts] = useState([]);
  const [loading, setLoading] = useState(true);
  const [error, setError] = useState<string | null>(null);

  useEffect(() => {
    setLoading(true);
    axios
      .get("/api/products.json")
      .then((res) => setProducts(res.data))
      .catch((err) => setError(err.message))
      .finally(() => setLoading(false));
  }, []);

  return { products, loading, error };
};
```

### Use the Hook in a Component

Now you can use your custom hook in any React component to display products:

```ts [src/components/ProductList.tsx]
import { useProducts } from "@/hooks/useProducts";

export const ProductList = () => {
  const { products, loading, error } = useProducts();

  if (loading) return <div>Loading...</div>;
  if (error) return <div>Error: {error}</div>;

  return (
    <ul>
      {products.map((p) => (
        <li key={p.id}>{p.name}</li>
      ))}
    </ul>
  );
}
```

## Summary

::callout{icon="i-lucide-lightbulb" color="info"}
The example above demonstrates how to fetch product data using a custom React hook.  
For best practices, create separate hooks for each type of API request (such as categories, orders, reviews, etc.) to keep your codebase clean, modular, and scalable.
::

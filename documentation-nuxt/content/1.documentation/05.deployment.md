Choose one of the following methods to deploy your project:
manual deployment from the .next folder, or automated deployment using platforms like Vercel.

## Method 1: Deploy from .next Folder (Manual/Custom Server)

::steps

### Build the project

Before deploying, you need to create an optimized production build of your project.
Run the following command in your terminal:

```bash [Terminal]
npm run build
```

Deploy your project to hosting platforms and go live.

## Method 2: Deploy with Vercel or Similar Platforms

::steps

### Push to Git Repository

Push your project to a Git provider:

- [GitHub](https://github.com/)
- [GitLab](https://gitlab.com/)
- [Bitbucket](https://bitbucket.org/)

### Connect to Hosting Platform

Connect your repository to a deployment platform:

- [Vercel (Recommended)](https://vercel.com/docs/concepts/projects/overview)
- [Netlify](https://docs.netlify.com/integrations/frameworks/next-js/overview/)
- [Render](https://render.com/docs/deploy-nextjs)
- [Firebase Hosting](https://firebase.google.com/docs/hosting/nextjs)

### Deploy and Go Live

Follow the platform instructions to trigger the deployment process.  
Your project will be built and published automatically.

For video tutorials, see:  
[YouTube: Deploy Next.js to Vercel](https://www.youtube.com/results?search_query=deploy+next+js+to+vercel)
